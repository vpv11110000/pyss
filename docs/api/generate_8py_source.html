31  то в атрибут MED_VALUE записыватся функция lambda: med_value
32  Значение параметров med_value, first_tx,
33  если задан список или кортеж в modificatorFunc, будет игнорироваться.
34  modificatorFunc=None - функция модификации времени следующего формирования транзакта
35  (float f(owner,currenttime)).
36  По сути этот параметр определяет смещение от model.getCurTime()+значение(med_value)

38  то в атрибут MODIFICATOR записыватся функция lambda owner,currenttime: modificatorFunc
39  Если задан список или кортеж, то при каждом вызове функции модификации времени

41  Значение параметров med_value, first_tx, если задан список или кортеж, будет игнорироваться.
42  first_tx=0 - время формирования первого транзакта.
43  Если задано None и modificatorFunc не является списком или кортежем,
44  то значение вычисляется как случайное число,
45  сформированное по равномерному распределению на интервале
46  [0,"среднее время формирования транзактов (см. med_value)"
47  +"результат вызова функции модификации времени
48  следующего формирования транзакта (см. modificatorFunc)"]
49  Значение параметров med_value, first_tx,
50  если задан список или кортеж в modificatorFunc, будет игнорироваться.
51  max_amount=None - максимальное количество транзактов формируемых блоком.
52  Если параметр max_amount равен 0 или None,
53  то блок генерирует неограниченное число транзактов до завершения моделирования,
54  или количеством, равным размеру списка modificatorFunc (если задан)

72 между моментами поступления

92 bl[MAX_AMOUNT] - максимальное количество транзактов формируемых блоком

97 bl[ENABLED] - флаг, если True, то блок может формировать транзакты
98  если False, то блок не может формировать транзакты
99 
100 Может быть заменён на блок Test или Transfer.
